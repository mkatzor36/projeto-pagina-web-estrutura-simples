Comunicação Client x Server

  • O que é Client?

Em termos simples, um cliente na web é qualquer software ou dispositivo que solicita e recebe informações de um servidor web. É como se fosse um cliente pedindo um serviço a um restaurante (o servidor).

    Exemplos de clientes web:

  • Navegadores: Chrome e Firefox são exemplos comuns. Quando você digita um endereço na barra de endereços, o navegador (cliente) envia uma solicitação ao servidor web onde o site está hospedado;
  • Aplicativos móveis: Apps como Facebook, Instagram e bancos também são clientes web. Eles se conectam a servidores para buscar e exibir informações;
  • Bots: Programas automatizados que interagem com sites, como os utilizados para coletar dados ou realizar tarefas repetitivas.

    Como funciona a interação entre cliente e servidor:

  1. Solicitação: O cliente envia uma solicitação ao servidor (por exemplo, ao digitar um URL);
  2. Processamento: O servidor recebe a solicitação, processa a informação e gera uma resposta;
  3. Resposta: O servidor envia a resposta ao cliente, que a interpreta e exibe na tela.

Um diagrama de rede de computadores de clientes comunicando com um servidor através da Internet.

Exemplo prático: Quando você acessa o Google, seu navegador (cliente) envia uma solicitação ao servidor do Google. O servidor do Google processa essa solicitação e envia de volta o código HTML da página inicial do Google. Seu navegador interpreta esse código e exibe a página na sua tela.

Navegadores Web

Um navegador web, também conhecido como browser, é um programa de computador que permite aos usuários acessarem e visualizarem páginas da internet. É como uma porta de entrada para a vasta rede de informações disponível online. Através do navegador, você pode navegar por sites, pesquisar informações, assistir vídeos, fazer compras e muito mais.

Exemplo de um navegador (Lynx) exibindo a página inicial da Wikipédia em modo texto.

Fonte: Wikipédia

Quando você digita um endereço web (URL) na barra de endereço do navegador e pressiona Enter, ele envia uma solicitação a um servidor remoto. Esse servidor localiza a página solicitada e a envia de volta ao seu computador. O navegador, então, interpreta o código da página (HTML) e exibe o conteúdo na sua tela.

Aplicações Web

“Uma aplicação Web é um software que é executado em um navegador da Web. As empresas precisam trocar informações e prestar serviços remotamente. Elas usam aplicações Web para se conectar com os clientes de forma conveniente e segura. Os recursos mais comuns do site, como carrinhos de compras, pesquisa e filtragem de produtos, mensagens instantâneas e feeds de notícias de mídia social, são aplicações Web no seu design. Eles permitem a você acessar funcionalidades complexas sem instalar ou configurar softwares”. (AWS, 2023)

    Linguagens de Programação mais comuns no Client:

  • JavaScript: É a linguagem mais popular e amplamente utilizada no desenvolvimento front-end. Ela roda diretamente no navegador do usuário, permitindo criar interfaces interativas e dinâmicas.;
  • TypeScript: Uma superset do JavaScript, adicionando tipagem estática para melhorar a organização e a manutenção de grandes projetos. É amplamente utilizado em frameworks como Angular;
  • Cascading Style Sheets (CSS): Responsável por estilizar a aparência de uma página web, definindo cores, fontes, layouts e outros elementos visuais.

    Frameworks e Bibliotecas mais comuns no Client:

  • React: Uma biblioteca JavaScript desenvolvida pelo Facebook, utilizada para criar interfaces de usuário complexas e eficientes;
  • Angular: Um framework JavaScript completo, desenvolvido pelo Google, para construir aplicações web robustas e escaláveis;
  • Vue.js: Um framework JavaScript progressivo, conhecido por sua curva de aprendizado suave e flexibilidade;
  • Svelte: Um framework JavaScript que compila componentes diretamente em JavaScript puro, resultando em aplicações mais rápidas e eficientes.

  • O que é Server?

Imagine um servidor web como uma biblioteca gigante. Nessa biblioteca, estão armazenados todos os arquivos que compõem um site: textos, imagens, vídeos e códigos. Quando você digita um endereço na internet e pressiona Enter, seu navegador (o cliente) está, na verdade, enviando uma solicitação a essa biblioteca para pegar um livro específico (uma página web).

“Servidores Web permitem a troca de dados e serviços pela Internet. A arquitetura cliente-servidor é o mecanismo subjacente da Internet. Quando você visita um site ou aplicação, seu navegador (como cliente) solicita dados de um servidor remoto e exibe a resposta. Um servidor Web é um componente de software que fornece dados estáticos, como imagens, arquivos e texto, em resposta às solicitações do cliente." (AWS, 2023)

    Para que serve um servidor web?

  • Armazenar arquivos: Todos os arquivos que compõem um site, como HTML, CSS, JavaScript e imagens, são armazenados em um servidor web;
  • Processar solicitações: Quando um usuário solicita uma página, o servidor web procura pelo arquivo correspondente e o envia para o navegador;
  • Gerar conteúdo dinâmico: Além de servir arquivos estáticos, um servidor web pode gerar conteúdo dinâmico, como notícias em tempo real ou resultados de buscas, a partir de bancos de dados.

    Outros tipos de Servidores:

  • Arquivos: Armazena e compartilha arquivos entre diversos usuários em uma rede, ideal para empresas que precisam centralizar dados, como documentos, imagens e vídeos. Permite que múltiplos usuários acessem e editem os mesmos arquivos;
  • Firewall: Atua como um guardião da rede, monitorando e filtrando o tráfego de dados, protegendo a rede interna de ataques externos, como hackers e vírus. Permite definir regras para o tráfego de dados, bloqueando ou permitindo conexões específicas;
  • Streaming:Transmite dados de áudio e vídeo em tempo real através da internet, sendo utilizado por plataformas de streaming como Netflix, Spotify e Twitch. Permite que os usuários assistam a vídeos ou ouçam músicas sem a necessidade de baixar os arquivos;
  • E-mail: Recebe, armazena e envia e-mails, sendo essencial para o envio e recebimento de mensagens eletrônicas. Permite o uso de recursos como caixas de entrada, pastas, filtros e endereços de e-mail personalizados;

Hardware

Em termos simples, o hardware de um computador refere-se a todas as suas partes físicas. São os componentes tangíveis que você pode tocar, como a placa-mãe, o processador, a memória RAM, os discos rígidos e as placas de rede. Esses componentes trabalham em conjunto para executar as instruções de software e fornecer os serviços que esperamos de um computador.

Embora os componentes básicos de um servidor sejam semelhantes aos de um computador pessoal, existem diferenças cruciais. Servidores são projetados para operar continuamente (24 horas por dia, 7 dias por semana), sob alta carga e com alta disponibilidade, o que exige componentes mais robustos e confiáveis. Há a capacidade de adicionar mais recursos (como processadores, memória ou armazenamento) para atender ao crescimento da demanda. Componentes duplicados ou sistemas de backup garantem a continuidade do serviço em caso de falhas.

    Componentes Essenciais de Hardware em um Servidor:

  • Processador (CPU): O "cérebro" do servidor. Processa as instruções dos programas e realiza cálculos. Servidores geralmente utilizam processadores mais poderosos e eficientes energeticamente;
  • Memória RAM: A memória de curto prazo do servidor, onde os dados e programas em execução são armazenados temporariamente. Servidores geralmente possuem grandes quantidades de RAM para lidar com múltiplas tarefas simultaneamente;
  • Armazenamento: Discos rígidos (HDDs ) e unidades de estado sólido (SSDs) armazenam os dados do servidor, como sistemas operacionais, aplicativos e arquivos. Servidores frequentemente utilizam arranjos de discos para aumentar a capacidade e a redundância;
  • Placa-mãe: A placa-mãe conecta todos os componentes do servidor, fornecendo os trilhos de dados e energia para cada componente;
  • Fontes de Alimentação: Fornecem energia elétrica estável e redundante para o servidor;
  • Placa de Rede: Permite que o servidor se conecte a outras redes e dispositivos;
  • Sistema de Refrigeração: Mantém o servidor em uma temperatura adequada para evitar superaquecimento.

Software

Software, em termos simples, são programas de computador. Eles fornecem as instruções que o hardware precisa para executar tarefas, desde as mais básicas, como gerenciar arquivos, até as mais complexas, como executar aplicativos empresariais.

Se o hardware é o corpo físico de um servidor, o software é sua alma, dando-lhe vida e propósito. É o conjunto de instruções que o hardware executa para realizar tarefas específicas. No contexto de servidores, o software desempenha um papel fundamental na definição das funcionalidades e na gestão dos recursos do sistema.

  • Sistema Operacional: É o software básico que gerencia o hardware e fornece uma interface para que outros programas possam interagir com o hardware. Exemplos comuns em servidores incluem Linux (CentOS, Ubuntu, Debian) e Windows Server;
  • Aplicativos: São programas projetados para realizar tarefas específicas, como servidores web (Apache, Nginx), bancos de dados (MySQL, PostgreSQL), servidores de e-mail (Postfix, Sendmail) e softwares de virtualização (VMware, VirtualBox);
  • Ferramentas de Administração: Permitem aos administradores gerenciar o servidor de forma remota, configurar serviços, monitorar o desempenho e solucionar problemas. Exemplos incluem SSH, Telnet, ferramentas de monitoramento como Nagios e Zabbix.
